---
name: Source Reconnaissance Violation
description: Never build deliverables without exhausting the target entity's public context first
type: feedback
---

Never generate deliverables targeting an external entity without first exhausting their publicly available context.

**Why:** A deliverable built on incomplete source context is structurally invalid regardless of internal quality. The input that triggers the deliverable is never the complete context.

**How to apply:** Before any deliverable that targets a company, person, or organization -- map their methodology, stated values, public documentation, and operational philosophy. Source reconnaissance is a HARD GATE in Rule 2.
